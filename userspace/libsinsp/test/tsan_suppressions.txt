# ThreadSanitizer suppressions for libsinsp unit tests (USE_TSAN=ON).
#
# Run unit tests under TSAN with these suppressions (from repo root):
#
#   TSAN_OPTIONS="suppressions=$(pwd)/userspace/libsinsp/test/tsan_suppressions.txt" \
#     ./build/libsinsp/test/unit-test-libsinsp --gtest_brief=1
#
# Or use the CMake target: cmake --build build --target run-unit-test-libsinsp-tsan
#
# Well-known basis for these suppressions:
#
# 1) Folly ThreadLocal and hazptr: Folly's ThreadLocal (folly/detail/ThreadLocalDetail.h)
#    uses a global registry of per-thread entries (ThreadEntrySet, allId2ThreadEntrySets_);
#    the file documents TSAN and lock ordering for this machinery. Folly's hazard
#    pointer implementation (HazptrThrLocal.h) uses a thread-local cache (hazptr_tc,
#    "Thread cache of hazptr_rec-s"). ConcurrentHashMap uses hazptr, so thread_manager
#    tests that use it hit this TLS path. TSAN reports races on these Folly internals
#    despite the design being per-thread and properly synchronized.
#

# The patterns below match only these Folly TLS/registry internals, not application code.
race:folly::threadlocal_detail
race:folly::SingletonThreadLocal
race:folly::hazptr_tc_tls
race:folly::detail::StaticSingletonManager
race:StaticSingletonManager.cpp
race:ThreadLocalDetail.h
race:ThreadLocalDetail.cpp
race:folly::ThreadEntrySet
race:removeThreadEntryFromAllInMap
race:onThreadExit
